# Aurora World Engine
# 03_データ設計書

---

# 1. 設計思想

Aurora World Engine は

「世界を作る」

「世界を遊ぶ」

「世界を育てる」

ことを目的としている。

そのため各データは独立して管理し、
必要に応じて自由に組み合わせられる。

---

# 2. データ構造

Aurora World Engine は

Project

Scenario

Character

Persona

Session

の5つを中心に構成される。

```
Project
│
├─ Scenario
│
├─ Character
│
└────────────┐
             │
Persona      │
     │       │
     └── Session
```

Sessionは

Project

Scenario

Persona

Character

を利用して作成される。

---

# 3. Project

Projectは一つの世界を表す。

Projectは

・名前

・概要

・世界観

・背景画像

・Scenario一覧

・Project Character一覧

を持つ。

Projectはゲーム一本に相当する。

---

# 4. Scenario

Scenarioは物語の開始地点である。

一つのProjectに

複数保存できる。

Scenarioは

・タイトル

・開始内容

を持つ。

---

# 5. Character

Characterは

AIが演じる登場人物である。

Characterは

・名前

・説明

・画像

を持つ。

Characterは

Project Character

Library Character

を区別せず管理する。

必要に応じて

Projectへ追加できる。

---

# 6. Persona

Personaは

プレイヤー自身である。

Personaは

・名前

・設定

を持つ。

Session開始時に選択する。

Session中も変更可能。

---

# 7. Session

Sessionは

プレイデータである。

Sessionは

・Project

・Scenario

・Persona

・Character一覧

・Chat履歴

・最終更新日時

を保存する。

Sessionは

オートセーブされる。

---

# 8. Character Library

Character Libraryは

全Characterを管理する。

Characterは

Project Characterも含め

一元管理する。

Character Libraryでは

・検索

・タグ

・お気に入り

に対応予定。

---

# 9. データの流れ

CREATE

↓

Project作成

↓

Scenario作成

↓

Character作成

↓

PLAY

↓

Session作成

↓

Session保存

↓

続きから
