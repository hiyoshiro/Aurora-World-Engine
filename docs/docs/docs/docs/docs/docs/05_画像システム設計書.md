# Aurora World Engine
# 05_画像システム設計書

---

# 1. 目的

画像はAurora World Engineの没入感を高める重要な要素である。

画像は必須ではない。

画像が無くてもゲームは遊べる。

---

# 2. 使用する画像

Auroraでは

・Project画像

・Character画像

の2種類のみ使用する。

Persona画像はVersion1では使用しない。

---

# 3. Project画像

Project画像は

その世界全体を表す画像である。

用途

・Project一覧

・開始画面

・チャット背景

---

推奨サイズ

1080 × 1920

縦長

スマホ壁紙サイズ

---

# 4. Character画像

Character画像は

登場人物を表す画像である。

用途

・Character Library

・Project Character

・チャット画面

---

推奨サイズ

768 × 1280

縦長

立ち絵を想定する。

---

# 5. 画像登録

画像登録時は

必ずプレビュー画面を表示する。

登録前に

・切り抜き

・拡大

・縮小

・位置調整

を行える。

保存時に

Aurora推奨サイズへ変換する。

---

# 6. AI画像生成

画像が無い場合

AI画像生成を利用できる。

流れ

画像生成

↓

プレビュー

↓

必要なら再生成

↓

保存

---

# 7. プレイ画面

Project画像を背景に表示する。

Character画像は

現在メインとなるCharacterを表示する。

将来的には

登場人物に応じて

自動で切り替える。

---

# 8. Character Library

Character画像をカード表示する。

画像が無い場合

標準アイコンを表示する。

---

# 9. Project一覧

Project画像をカード背景として表示する。

画像が無い場合

Auroraロゴを表示する。
