# Aurora World Engine
# 06_アーキテクチャ設計書

---

# 1. 目的

この設計書は Aurora World Engine のシステム構成を定義する。

Aurora は

「長く育てられるAIストーリーゲームエンジン」

を目指す。

そのため

UI

AI

データ

を完全に分離する。

---

# 2. システム構成

Auroraは以下の4層で構成する。

```
Frontend

↓

Backend

↓

Aurora Engine

↓

Storage
```

必要に応じて

Aurora Engineから

OpenRouterへアクセスする。

---

# 3. Frontend

Frontendは

画面表示のみ担当する。

主な役割

・PLAY

・CREATE

・CHARACTER LIBRARY

・SETTINGS

・画像表示

・画像切り抜き

・スマホUI

Frontendには

ゲームロジックを書かない。

---

# 4. Backend

Backendは

画面とAurora Engineを接続する。

主な役割

・データ取得

・データ保存

・API

・画像保存

---

# 5. Aurora Engine

Aurora Engineは

ゲーム全体を管理する。

主な役割

・Project読込

・Scenario読込

・Character管理

・Persona管理

・Session管理

・AIプロンプト生成

・AI応答処理

Aurora Engineは

Aurora World Engine の心臓部である。

---

# 6. Storage

Storageは

データ保存のみ担当する。

保存対象

・Project

・Character

・Persona

・Scenario

・Session

・画像

Version1では

JSON保存

Version2以降で

SQLiteへ移行可能とする。

---

# 7. AI

AIは

Aurora GMとして動作する。

Aurora Engineのみが

AIと通信する。

Frontendから直接AIを呼ばない。

---

# 8. データの流れ

```
User

↓

Frontend

↓

Backend

↓

Aurora Engine

↓

OpenRouter

↓

Aurora Engine

↓

Session保存

↓

Frontend
```

---

# 9. 設計思想

各モジュールは

独立して動作する。

UIを変更しても

Aurora Engineは変更しない。

AIモデルを変更しても

UIは変更しない。

Storageを変更しても

ゲームロジックは変更しない。
